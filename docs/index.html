Configuration

The Doxygen Preview Extension searches in your Visual Studio solution directory for a XML file named doxygen_config.xml. The content of the file should look like this.

<config>
<source>https://rabbid76.github.io/VSIX-Doxygen-Extension/</source>
<configurationfile>.\DocsMake\DocsMake.txt</configurationfile>
</config>

The source tag contains the location of your doxygen documentation. The configurationfile tag contains the absolute or relative path to the doxygen configuration file.

Doxygen configuration file

The following attributes are of the doxygen configuation file are considerd by the Doxygen Preview Extension:

  • OUTPUT_DIRECTORY
    The OUTPUT_DIRECTORY tag is used to specify the (relative or absolute) path into which the generated documentation will be written. If a relative path is entered, it will be relative to the location where doxygen was started. If left blank the current directory will be used.
  • IMAGE_PATH
    The IMAGE_PATH tag can be used to specify one or more files or directories that contain images that are to be included in the documentation (see the \image command).
  • GENERATE_HTML
    If the GENERATE_HTML tag is set to YES, doxygen will generate HTML output The default value is: YES.
  • HTML_OUTPUT
    The HTML_OUTPUT tag is used to specify where the HTML docs will be put. If a relative path is entered the value of OUTPUT_DIRECTORY will be put in front of it.
    The default directory is: html. This tag requires that the tag GENERATE_HTML is set to YES.
  • MSCGEN_PATH
    You can define message sequence charts within doxygen comments using the \msc command. Doxygen will then run the mscgen tool (see: [http://www.mcternan.me.uk/mscgen/])) to produce the chart and insert it in the documentation. The MSCGEN_PATH tag allows you to specify the directory where the mscgen tool resides. If left empty the tool is assumed to be found in the default search path.
  • HAVE_DOT
    If you set the HAVE_DOT tag to YES then doxygen will assume the dot tool is available from the path. This tool is part of Graphviz (see: [http://www.graphviz.org/]), a graph visualization toolkit from AT&T and Lucent Bell Labs. The default value is: NO.
  • DOT_PATH
    The DOT_PATH tag can be used to specify the path where the dot tool can be found. If left blank, it is assumed the dot tool can be found in the path. This tag requires that the tag HAVE_DOT is set to YES.
  • PLANTUML_JAR_PATH
    When using plantuml, the PLANTUML_JAR_PATH tag should be used to specify the path where java can find the plantuml.jar file. If left blank, it is assumed PlantUML is not used or called during a preprocessing step. Doxygen will generate a warning when it encounters a \startuml command in this case and will not generate output for the diagram.
  • PLANTUML_JAR_PATH
    When using plantuml, the specified paths are searched for files specified by the !include statement in a plantuml block.

Add an image to source code documentation

To embedded an image use the special doxygen command @image.

e.g.

/ ****************************************************************** // **
\brief Foo
\image html my_image.svg "my image caption"
\author anyone
\date 01.01.2017
\version 1.0
/
void Foo( void )
{
...
};

If Doxygen Preview Extension is installed the image will be displayed in the quicktip of the special doxygen command @image html my_image.svg.
In order for the images to be displayed, the directories indicated in the IMAGE_PATH attribute of the Doxygen configuration file are analyzed. All image file names are managed in a dictionary cache. If a new image is added, the cache has to be updated:
Extras -> Options -> Doxygen Preview -> Doxygen configuration -> Update Cache.

Create Dot Graph

cf Gaphviz - Graph Visualization Software [http://www.graphviz.org/]
cf Doteditor - [http://vincenthee.github.io/DotEditor/]

\dot
digraph "UML diagram"
{
// INTERACTIVE_SVG=YES
edge [fontsize="10", style="solid", fontname="Helvetica"]
node [shape=record, fontname="Helvetica", fontsize="10"];
// inheritance
edge [dir="back", style="solid", arrowhead="none", arrowtail="onormal", color="midnightblue"]
A -> B1
A -> B2
A -> B3
// association
edge [dir="front", style="dashed", arrowhead="vee", arrowtail="none", color="grey25"]
B1 -> C1 [label="assoziation", headlabel="*", taillabel="1"]
// aggregation
edge [dir="back", style="solid", arrowhead="none", arrowtail="odiamond", color="grey25"]
B2 -> C2 [label="aggregation", headlabel="*", taillabel="1"]
// composition
edge [dir="back", style="solid", arrowhead="none", arrowtail="diamond", color="grey25"]
B3 -> C3 [label="composition", headlabel="*", taillabel="1"]
}
\enddot
